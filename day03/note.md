1. temperature/top_p/max_tokens 分别干什么？Agent 开发中怎么设？
采样参数：temperature 控随机性（0确定，1创意），
top_p 控候选池（一般保持0.9或1），max_tokens 控输出长度。
Agent开发中工具调用用0，对话用0.7。
2. messages 三种角色的区别？多轮对话的本质是什么？
消息角色：system 定规则，user 提问题，assistant 给回复。
多轮对话的本质 = 不断拼接历史消息，模型本身无状态，全靠你把整个对话流水账每次完整发过去。
3. 对话越来越长会有什么问题？滑动窗口截断的优缺点？
长对话问题：成本飙升、超窗失忆、性能下降。
滑动窗口（保留最近N条）简单可控，但代价是永久丢失早期信息（AI失忆）。
4. 如果让你设计一个生产级对话系统，你会怎么管理对话历史？
生产级方案：采用三层记忆架构——
短期：最近10-20轮（Redis/内存，滑动窗口）
中期：被移出的对话生成摘要（数据库）
长期：完整对话向量化后存入向量库（RAG检索）
每次请求 = 短期窗口 + 相关摘要 + 检索到的历史片段，兼顾成本与记忆。
